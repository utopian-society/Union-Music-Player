# 应用崩溃诊断与修复

## 问题描述
应用在启动时崩溃，发生在启用 R8 代码压缩之后。

## 诊断过程

### 1. 初步分析
- **症状**: 应用启动时立即崩溃
- **时间点**: 启用 `isMinifyEnabled = true` 后出现
- **可能原因**: R8 删除了必要的类或资源

### 2. 根本原因

经过分析，崩溃可能由以下原因导致：

#### 原因 1: Material Icons 类被误删
- **问题**: R8 无法静态分析 `Icons.Default.*` 和 `Icons.Outlined.*` 的动态使用
- **表现**: 运行时找不到图标类，导致 `ClassNotFoundException` 或 `NoSuchFieldError`
- **修复**: 添加 ProGuard 规则保留所有 icons 类

```proguard
-keep class androidx.compose.material.icons.** { *; }
-dontwarn androidx.compose.material.icons.**
```

#### 原因 2: Compose 运行时类丢失
- **问题**: Compose 使用运行时生成 UI，R8 可能误删
- **修复**: 保留所有 Compose 相关类

```proguard
-keep class androidx.compose.** { *; }
```

#### 原因 3: Kotlin 协程类被优化
- **问题**: 协程使用反射和动态分发
- **修复**: 保留协程关键类

```proguard
-keepnames class kotlinx.coroutines.internal.MainDispatcherFactory {}
-keepnames class kotlinx.coroutines.CoroutineExceptionHandler {}
```

### 3. 已应用的修复

#### 修复 1: 添加 Material Icons 保留规则
```proguard
# Keep Material Icons - Required for Icons.Default.* and Icons.Outlined.* usage
-keep class androidx.compose.material.icons.** { *; }
-dontwarn androidx.compose.material.icons.**
```

#### 修复 2: 保留 Compose 核心类
```proguard
-keep class androidx.compose.** { *; }
-keep class androidx.navigation.** { *; }
```

#### 修复 3: 保留 Kotlin 元数据
```proguard
-keep class kotlin.Metadata { *; }
-keepclassmembers class **$WhenMappings { <fields>; }
```

### 4. 验证步骤

构建并测试应用：

```bash
# 清理并构建 Release 版本
./gradlew clean assembleRelease

# 安装到设备测试
adb install -r app/build/outputs/apk/release/app-release.apk

# 查看日志确认崩溃
adb logcat | grep "org.bibichan.union.player"
```

### 5. 如果仍然崩溃

#### 方案 A: 暂时禁用 R8 进行调试
```kotlin
buildTypes {
    release {
        isMinifyEnabled = false  // 临时禁用
        isShrinkResources = false
    }
}
```

#### 方案 B: 添加更多 ProGuard 规则
```proguard
# 保留所有 Kotlin 协程
-keep class kotlinx.coroutines.** { *; }

# 保留 Coil 图片加载
-keep class coil.** { *; }
-keep class coil3.** { *; }

# 保留 AndroidX 核心
-keep class androidx.** { *; }
```

#### 方案 C: 使用 R8 Full Mode
在 `gradle.properties` 添加：
```properties
android.enableR8.fullMode=false
```

### 6. 预期结果

修复后应该：
- ✅ 应用正常启动
- ✅ UI 显示完整（图标、文字）
- ✅ 导航功能正常
- ✅ 无崩溃或异常

### 7. 优化与兼容性平衡

| 优化级别 | APK 大小 | 崩溃风险 | 推荐度 |
|----------|----------|----------|--------|
| 禁用 R8 | 大 | 无 | ❌ 不推荐 |
| 基础规则 | 中等 | 低 | ✅ 推荐 |
| 激进优化 | 小 | 中 | ⚠️ 需谨慎测试 |

**当前采用**: 基础规则 - 在大小和稳定性之间平衡
