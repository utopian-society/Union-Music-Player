# APK 体积优化计划 - Dex 文件过多问题

## 问题分析

### 当前状态
- **项目类型**: Android 音乐播放器 (Jetpack Compose + Kotlin)
- **编译 SDK**: 36
- **最低 SDK**: 24
- **主要依赖**: Compose UI, Material3, Coil, Navigation, JAudioTagger (遗留引用)

### 发现的核心问题

1. **代码压缩未启用**
   - `isMinifyEnabled = false` in release build
   - R8/ProGuard 未运行，所有代码完整打包

2. **ProGuard 规则过于宽松**
   - `-keep class org.jaudiotagger.** { *; }` 保留整个库
   - `-keep class com.mpatric.mp3agic.** { *; }` 保留整个库
   - 这些库可能未被实际使用（代码搜索未发现导入）

3. **Compose 依赖冗余**
   - `material-icons-extended` 包含 2000+ 图标，体积大
   - 可能只使用了少数图标

4. **Debug 符号保留过多**
   - `-keepattributes SourceFile,LineNumberTable` 增加体积

5. **可能的遗留依赖**
   - README 提到 ExoPlayer/Media3，但 build.gradle.kts 中未发现
   - JAudioTagger 和 mp3agic 在 proguard 中保留，但代码中可能未使用

## 优化策略

### 1. 启用 R8 代码压缩 (预期减少 30-50%)
```kotlin
buildTypes {
    release {
        isMinifyEnabled = true
        isShrinkResources = true
        proguardFiles(...)
    }
}
```

### 2. 优化 ProGuard 规则 (预期减少 10-20%)
- 移除未使用库的 keep 规则
- 使用 `-dontwarn` 代替 `-keep` 对于可选依赖
- 让 R8 自动优化第三方库

### 3. 依赖优化 (已尝试)
- 尝试将 `material-icons-extended` 替换为 `material-icons-core`
- **结果**: 代码使用了多个 extended 图标 (Album, SkipPrevious, LibraryBooks 等)
- **决定**: 保留 extended 库以确保功能完整性
- **未来优化**: 可以使用自定义图标子集或 SVG 图标

### 4. 资源压缩 (预期减少 5-10%)
- 启用 `isShrinkResources = true`
- 移除未使用的 drawable 和 mipmap

### 5. Dex 优化
- R8 自动进行 dex 合并和优化
- 使用 `--min-api` 匹配 minSdk 24
