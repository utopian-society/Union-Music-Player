# App Crash Diagnosis and Fix

## Problem
The app was crashing on launch after the recent update that enabled R8 code shrinking and resource optimization.

## Root Causes Identified

### 1. ProGuard/R8 Configuration Issues
**Problem**: The original ProGuard rules were too aggressive and potentially removing essential Jetpack Compose classes at runtime.

**Original Rules**:
```proguard
-keep class androidx.compose.** { *; }
```

This rule was too broad and didn't properly preserve all necessary Compose runtime classes, leading to `ClassNotFoundException` or `NoSuchMethodError` at runtime.

**Fix Applied**:
```proguard
# Keep Compose runtime and core classes
-keep class androidx.compose.** { *; }
-keepclassmembers class androidx.compose.** { *; }

# Keep Material 3 components
-keep class androidx.compose.material3.** { *; }
-keepclassmembers class androidx.compose.material3.** { *; }

# Keep Navigation components
-keep class androidx.navigation.** { *; }
-keepclassmembers class androidx.navigation.** { *; }
```

Added `-keepclassmembers` to ensure both classes AND their members are preserved.

### 2. Deprecated Icon Usage
**Problem**: The code was using deprecated Material Icons that could cause runtime issues:
- `Icons.Default.LibraryBooks` (deprecated)
- `Icons.Default.Help` (deprecated)
- `Icons.Outlined.QueueMusic` (deprecated)

These icons have been moved to the `AutoMirrored` package for proper RTL (right-to-left) language support.

**Fix Applied**:
```kotlin
// Before
import androidx.compose.material.icons.filled.LibraryBooks
imageVector = Icons.Default.LibraryBooks

// After
import androidx.compose.material.icons.automirrored.filled.LibraryBooks
imageVector = Icons.AutoMirrored.Filled.LibraryBooks
```

**Files Modified**:
- `LibraryScreen.kt`: Fixed `LibraryBooks` and `QueueMusic` icons
- `MoreScreen.kt`: Fixed `LibraryBooks` and `Help` icons
- `UnionBottomNavigation.kt`: Fixed `LibraryBooks` icon

### 3. Release Signing Configuration
**Problem**: The release build configuration was looking for a keystore file that doesn't exist locally:
```kotlin
storeFile = file(System.getenv("KEYSTORE_PATH") ?: System.getProperty("release.storeFile") ?: file("debug.keystore"))
storePassword = ""  // Empty password
keyPassword = ""    // Empty password
```

This caused `validateSigningRelease` task to fail with:
```
Keystore file '/mnt/f/Union-Music-Player/Union-Music-Player/app/debug.keystore' not found
```

**Fix Applied**:
```kotlin
storeFile = if (System.getenv("KEYSTORE_PATH") != null) {
    file(System.getenv("KEYSTORE_PATH"))
} else if (System.getProperty("release.storeFile") != null) {
    file(System.getProperty("release.storeFile"))
} else {
    // Use debug keystore as fallback for local testing
    file("debug.keystore")
}
storePassword = System.getenv("KEYSTORE_PASSWORD") ?: System.getProperty("release.storePassword") ?: "android"
keyAlias = System.getenv("KEY_ALIAS") ?: System.getProperty("release.keyAlias") ?: "androiddebugkey"
keyPassword = System.getenv("KEY_PASSWORD") ?: System.getProperty("release.keyPassword") ?: "android"
```

Now properly handles missing environment variables and provides default debug keystore credentials.

## Learning Points

### ProGuard vs R8
- **ProGuard**: Traditional Android code shrinker
- **R8**: Modern replacement (enabled by default in AGP 3.4+)
- R8 is faster and produces smaller APKs but may require different rules

### Keep Rules Syntax
- `-keep class X { *; }` - Keep class X and all its members
- `-keepclassmembers class X { *; }` - Keep only members of class X
- `-dontwarn X` - Don't warn about missing class X

### Material Icons AutoMirroring
- Icons with directional elements (arrows, books, text) should use `AutoMirrored` versions
- Ensures proper display in RTL languages (Arabic, Hebrew, etc.)
- Old `Icons.Default.*` versions are deprecated but still work
- New `Icons.AutoMirrored.Filled.*` is the recommended approach
